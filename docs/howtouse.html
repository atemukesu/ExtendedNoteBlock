Extended Note Block 使用教程

前置要求

在启动游戏前，请务必确保您的计算机已正确安装 FFmpeg 和 FluidSynth，并已将它们配置到系统的环境变量中。配置好环境变量之后，模组将会自动检测并填写路径，无需您主动填写。这两个工具是生成音频资源所必需的，缺少它们将导致本模组无法正常工作。

安装与配置 FFmpeg

Windows 系统

  1. 访问 FFmpeg 官方网站，在 Windows 图标下找到推荐的下载链接（如 gyan.dev 或 BtbN）。
  2. 下载最新版本的压缩包（通常为 `.zip` 或 `.7z` 格式）并解压。
  3. 将解压后的文件夹移动到一个简洁的路径，例如 C:\ffmpeg。注意：路径中请勿包含中文或空格，以防意外错误。
  4. 记住该文件夹中 bin 目录的完整路径，例如 C:\ffmpeg\bin。
  5. 配置环境变量：
    • 右键点击“此电脑” → “属性” → “高级系统设置” → “环境变量”。
    • 在“系统变量”区域中，找到并选中名为“Path”的变量，然后点击“编辑”。
    • 在编辑窗口中，点击“新建”，然后粘贴您刚刚记住的 bin 目录路径 (C:\ffmpeg\bin)。
    • 一路点击“确定”保存所有设置。
  6. 验证安装：
    • 按下 Win + R 键，输入 cmd 并回车，打开命令提示符。
    • 输入 ffmpeg -version 并回车。如果显示出版本信息而非错误提示，则表示安装成功。

macOS 系统

  1. 首先，安装 Homebrew（若尚未安装）：
    • 打开“终端”应用。
    • 粘贴并执行命令：/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    • 根据提示完成安装过程。
  2. 在终端中执行 brew install ffmpeg 命令安装 FFmpeg。
  3. 验证安装：输入 ffmpeg -version，看到版本信息即表示成功。

Linux 系统

  1. 打开终端。
  2. 根据您的发行版，执行相应的安装命令：
    • Ubuntu/Debian: sudo apt update && sudo apt install ffmpeg
    • Fedora/RHEL: sudo dnf install ffmpeg
    • Arch Linux: sudo pacman -S ffmpeg
  3. 验证安装：输入 ffmpeg -version，看到版本信息即表示成功。

安装与配置 FluidSynth

Windows 系统

  1. 访问 FluidSynth 的 GitHub 发布页。
  2. 在最新的发布版本下，找到并下载适用于 Windows 的压缩包。
  3. 解压并将其移动到简洁的路径，例如 C:\fluidsynth。
  4. 按照与 FFmpeg 完全相同的步骤，将 FluidSynth 的 bin 目录路径（例如 C:\fluidsynth\bin）添加到系统的“Path”环境变量中。
  5. 验证安装：
    • 打开新的命令提示符窗口。
    • 输入 fluidsynth --version 并回车。显示版本信息即表示安装成功。

macOS 系统

  1. 打开终端，确保已安装 Homebrew。
  2. 执行命令：brew install fluidsynth。
  3. 验证安装：输入 fluidsynth --version。

Linux 系统

  1. 打开终端。
  2. 根据您的发行版，执行相应的安装命令：
    • Ubuntu/Debian: sudo apt install fluidsynth
    • Fedora/RHEL: sudo dnf install fluidsynth
    • Arch Linux: sudo pacman -S fluidsynth
  3. 验证安装：输入 fluidsynth --version。

渲染核心数设置

在模组的配置界面中，您需要设置用于音频渲染的线程数量。以下是一些通用建议：

  • 建议设置为您计算机 CPU 逻辑核心数的一半左右，但不超过总核心数。
  • 对于 4 核 CPU，建议设置为 2-3。
  • 对于 8 核 CPU，建议设置为 4-6。
  • 设置过高的线程数可能会加快渲染速度，但可能导致系统卡顿。
  • 设置过低的线程数会减慢渲染速度，但能保证系统稳定性。

首次加载时，模组需要进行音频资源渲染，这可能需要数分钟到数十分钟不等，具体取决于您的 CPU 性能和线程数设置，请耐心等待。渲染完成后，即可正常使用。
